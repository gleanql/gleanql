# GleanQL gap audit — 2026-07-02

60-agent audit at v0.1.18 (8 dimensions, every finding adversarially verified: 50 confirmed, 1 refuted). Line numbers reference the working tree at commit 6be0ba2.

## Needs documentation

1. **Server mutation primitive has zero docs anywhere** — `serverMutate` (packages/vite/src/types.ts:136), `runServerMutation` (packages/client/src/mutation.ts:143), and `integration.mutate` (packages/client/src/integration.ts) shipped in 0.1.9/0.1.10 and are announced only in CHANGELOG.md; grep finds no hits in docs/*.md, any README, or examples/. It is the only plugin option undocumented project-wide (docs/vite.md options table lines 44-57 and docs/api.md:77 both omit it).
2. **Auto-injected `userErrors` selection on mutation payloads is undocumented** — since 0.1.11, `injectUserErrors` (packages/compiler/src/selector-compile.ts:167, invoked at :138) adds `userErrors { field message code }` to every mutation-root payload that has the field. docs/runtime.md:296 and docs/usage.md:255 describe userErrors on state but never say it's injected; docs/core.md's "Identity injection" section covers only `__typename`/`id`.
3. **`@gleanql/client/testing` only resolves after the vite plugin provisions the shadow package** — packages/client/package.json exports only "."; the subpath is created by generate.ts:249. docs/testing.md (lines 26/70/98/139) tells users to import it after just `pnpm add -D vitest`, so a fresh install fails with ERR_PACKAGE_PATH_NOT_EXPORTED; the plugin prerequisite is mentioned only for mutation/subscription binding (lines 126-130).
4. **FrameworkPreset hot-swap/dep-optimizer hooks are undocumented** — `viteConfigPatch`, `operationsDigest`, `volatileModules`, `hotUpdateEvent` (packages/vite/src/types.ts:73-106, the source's self-described "footgun" area) are absent from docs/vite.md's preset table (lines 108-115), which explicitly invites custom preset objects; vite.md:70 still describes the pre-0.1.3 "invalidates and reloads the page" behavior.
5. **Compiler diagnostics table lists 4 of 7 codes** — docs/compiler.md (lines 221-226) omits `unsupported-list-flow`, `unawaited-deferred-read` (prose-only at line 153), and `imported-helper` (documented nowhere) vs packages/compiler/src/diagnostics.ts:7-14.
6. **examples/storefront and examples/storefront-fixture have no README and are missing from the root README examples table** — storefront is the acceptance harness `pnpm example` runs (contradicting CONTRIBUTING.md:25's "Bootable apps"); storefront-fixture is the shared schema/transport both featured apps depend on (rwsdk-real/vite.config.mts:19, remix-real/vite.config.ts:28) and is undocumented repo-wide.
7. **starter's "copy this directory to start" breaks standalone** — examples/starter/package.json pins `@gleanql/client`/`@gleanql/vite` to `workspace:*`; no doc tells copiers to swap in the published 0.1.18 versions (README.md:80 and docs/get-started.md:97 repeat the advice).
8. **File-derived operation names for anonymous default-export handlers (0.1.7) documented nowhere** — `fileDerivedComponentName` (packages/compiler/src/ast-facade.ts:92) names operations like OrdersCreate from the file path; absent from docs/compiler.md and docs/testing.md's naming section, and 0.1.7 has no changelog entry either. Matters more since 0.1.15 promotes exactly these handlers (docs/runtime.md:189).
9. **Read-masking has one option-table row and no usage guidance** — docs/vite.md:56 only; absent from README's feature list, usage.md's "Handle errors" table, and runtime.md; no example of a violation warning or fix (rwsdk-real does enable `masking: true`).
10. **Contributor-facing test/deploy split undocumented** — root `pnpm example` covers only the two storefront harness workspaces; rwsdk-real's harness.test.tsx is invisible to the root glob (`.test.ts` only, vitest.config.ts:16); CONTRIBUTING.md never mentions the auto-deploy pipeline (.github/workflows/deploy-docs.yml, Cloudflare secrets, generated site files) that docs/README.md only partially covers.

## Inconsistencies / judgement calls

1. **`findUseMutationSites`/`UseMutationSite` back-compat shim has zero callers repo-wide** (packages/compiler/src/mutation-binding.ts:59, 109-112, publicly exported) — remove on a 0.1.x package, or keep for published consumers and document both names; either way docs/api.md currently documents only the legacy name.
2. **@gleanql/compiler exact-pins a month-old nightly `@typescript/native-preview@7.0.0-dev.20260605.1` as optionalDependency** — impact is narrow (@gleanql/vite bundles compiler as a devDep, so typical users never install it), but direct compiler installers download platform binaries by default, and @gleanql/vite users opting into `backend: "tsgo"` must install the nightly themselves — no doc says so (docs/vite.md:89 covers only the fallback).
3. **@gleanql/vite declares no vite peer range and no doc states supported versions** — deliberate structural typing (types.ts:205-238) lives only in source comments; either an optional peer range (peerDependenciesMeta) or a one-line compatibility note in docs/vite.md.
4. **remix-real and rwsdk-todo have zero automated tests; the consumer-harness e2e runs on one CI leg only** — CI gives both apps build+typecheck only; rwsdk-real/tests/harness.test.tsx (the only @gleanql/client/testing e2e) runs solely via the ubuntu/node-22 examples job, never on node-20/windows or local `pnpm test`. Add smoke tests / widen the glob, or document the split as intentional.
5. **@gleanql/compiler and @gleanql/codegen ship src/ to npm with no consumer** — `files: ["dist", "src"]` mirrors client/core, but only those two need src for vite provisioning (provision.ts); compiler's 144K + codegen's 28K src are dead tarball weight, no sourcemaps reference them. Trim, or keep for uniformity.
